# NCR Status Message Reference Extracted

## Purpose
This file captures the operational decoding rules extracted from the provided NCR status-message reference so they can be used directly in EJLive development.

## 1. Base NCR status message shape
The core NCR journal/status pattern is:

```text
tttt * m * d * aa.a, M-bb, R-c
```

Meaning:
- `tttt`: transaction serial number
- `m`: message status type
  - `1` = unsolicited status generated by the ATM itself
  - `2` = solicited status returned in response to a host/network action
- `d`: device code
- `aa...`: transaction or device status field
- `M-bb`: module/diagnostic status
- `R-c...`: supply/bin/cassette status

## 2. Device code map
The extracted document confirms at least these device codes:
- `A` = Time of Day Clock
- `B` = Power Failure
- `D` = Card Reader / Writer
- `E` = Currency Dispenser
- `F` = Depository
- `G` = Receipt Printer
- `H` = Journal Printer
- `L` = Encryptor
- `P` = Sensors
- `V` = Statement Printer

## 3. Device-specific decoding rules

### 3.1 Device `D` — Card Reader / Writer
#### Transaction/device status (`a`)
- `0` = minor problem, inspect module and supply fields
- `1` = customer did not take card in time; card captured or jammed
- `2` = eject failure; card captured or jammed
- `3` = failed to write Track 3
- `4` = invalid Track 3 data sent from network

#### Module status (`M-bb`)
Important values extracted:
- `00` success
- `03` read error
- `04` write error
- `05` no card in reader at start of command
- `07` shutter jammed open
- `10` shutter jammed closed
- `11` card jammed
- `12` card captured after unsuccessful eject
- `13` possible customer tampering
- `18` card jam during capture
- `20` device still inoperative
- `37` card return on power failure not supported
- `38` card capture bin full
- `39` card capture bin shutter mechanism failure
- `40` card capture bin removed or not configured
- `41` card return on power failure failed
- `146` SDC link failure

#### Supply status (`R-c`)
- `0` = no new state
- `1` = card capture bin is not overfilled
- `4` = card capture bin is overfilled

### 3.2 Device `E` — Currency Dispenser
#### Transaction/device status (`aaaaaaaaa`)
The first digit is the **dispense outcome code**:
- `0` = dispense successful but an exception occurred
- `1` = fewer bills dispensed than requested
- `2` = no bills dispensed
- `3` = faulty dispense; dispensed quantity unknown
- `4` = no bills dispensed or card was not ejected before bills were dispensed
- `5` = some/all bills not taken and retracted into reject bin

Remaining digits encode dispensed note counts by cassette.

#### Module status (`M-bb`)
Important values extracted:
- `00` no error
- `01` pick requested from cassette not installed
- `02` too many bills rejected
- `03` pick failure
- `04` pick failure from low/out cassette
- `05` sensor failure or currency jam
- `06` divert gate failure
- `07` reject bin not present
- `08` reject bin overfilled or blocked
- `09` communications failure at dispenser
- `10` operation not attempted due to previous failure
- `11` bills still in transport
- `12` presenter mechanism failed/jammed
- `13` exit shutter jammed open
- `14` exit shutter jammed closed
- `18` currency jam in presenter transport or sensor failed
- `19` exit sensor blocked
- `20` dispenser not configured
- `21` security violation
- `22` security inhibition
- `28` NVRAM access issue
- `29` presenter transport timing or motor failure
- `30` presenter transport sensors blocked/failed
- `32` execution processor heartbeat failed
- `33` I2C bus communication failure
- `34` present/retract attempted with no notes staged/presented
- `35` power/interlock interruption during operation
- `36` pick valve test not attempted because cassettes were installed
- `37` tamper-indicating service ON
- `38` tamper-indicating service OFF
- `39` LVDT or associated circuitry malfunction
- `40` note parameters not configured or NVRAM corruption/first power-up
- `41` pick attempted from cassette disabled for in-service replenishment
- `146` SDC link failure

#### Supply status (`R-ccccc`)
- character 1 = divert bin state
  - `1` not overfilled
  - `4` overfilled
- characters 2–5 = cassette supply state for cassettes 1–4
  - `0` no new state
  - `1` sufficient
  - `2` low
  - `3` out

### 3.7 Device `P` — Sensors
The document defines this as a multi-character status field describing:
- tamper/alarm/mode-switch change
- mode switch normal/supervisor
- vibration/heat, door, silent alarm, electronics enclosure
- deposit bin/card capture bin/reject bin in/out
- cassette positions 1–4 in/out

This means the parser should treat device `P` as a **sensor state vector**, not a simple scalar code.

## 4. Command rejects
A separate NCR pattern exists for command reject lines:

```text
*tttt*2*CNN
```

Where `C` is the command reject category and `NN` refines the reason.

Important categories:
- `1` = message authentication code
- `2` = time variant number failure
- `3` = security terminal number mismatch
- `A` = message format error
- `B` = field value error
- `C` = illegal message type for current mode
- `D` = hardware failure
- `E` = not supported

Important examples:
- `*1234*2*D01` = encryption failure during key change message
- `C01` = message only accepted while ATM is in-service
- `C03` = not accepted while out-of-service/supply mode
- `E01` = command not supported by current software version
- `E02` = required device not configured

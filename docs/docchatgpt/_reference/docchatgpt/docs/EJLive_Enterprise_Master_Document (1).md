## 1. ملخص شامل للنظام

### 1.1 تعريف النظام

**EJLive Enterprise** هو نظام مكتبي متكامل (Desktop Application) مبني بلغة C# / .NET Framework 4.8 (Windows Forms)، مصمم خصيصًا لـ:

- **أرشفة** الجورنال الإلكتروني (Electronic Journal) لأجهزة الصراف الآلي (ATM) في الوقت الفعلي
- **مراقبة** حالة الصرافات ومستوى خدمتها مركزيًا
- **التحكم عن بُعد** بالصرافات (إعادة تشغيل، تغيير كلمة السر، إرسال صور، الوصول كشبح)
- **تحليل** عمليات الجورنال المالية واستخراج تقارير احترافية
- **إدارة** شبكة الصرافات كاملة من نقطة مركزية واحدة

### 1.2 الأنواع المدعومة من الصرافات

| النوع | آلية الجورنال | مسار الملفات | ملاحظات |
|-------|--------------|-------------|---------|
| **NCR** | ملف واحد ثابت يُحدَّث بالكتابة فوق نفسه (Overwrite Mirroring) | `C:\NCRJournal\` | 3 ملفات: `EJDATA.LOG`, `EJRCPY.LOG`, `EJDATA.LOb` |
| **GRG** | ملفات يومية متعددة | `D:\GRGData\EJ\` | صيغة: `EJ_*.dat` و `TRACE*` |
| **Wincor Nixdorf (WN)** | ملفات يومية | `C:\WOSA\EJ\` | صيغة: `*.ej`, `*.log` |

### 1.3 بيئة التشغيل

- **نظام التشغيل:** Windows 7 SP1 أو أحدث (Windows 10 مدعوم بالكامل)
- **إطار العمل:** .NET Framework 4.8
- **بيئة التطوير:** Visual Studio 2019/2022
- **قاعدة البيانات:** SQLite (محلي) — مقترح الترقية إلى PostgreSQL/SQL Server
- **البورت الافتراضي:** TCP/5656
- **التشفير:** AES-256-CBC
- **الضغط:** Deflate Algorithm
- **التحقق من السلامة:** MD5 + SHA-256 + Checksum

## 3. المكونات والخدمات التفصيلية

### 3.1 محركات النواة (EJLive.Core.Engine)

#### أ) FileWatcherEngine (524 سطر)
**الوظيفة:** مراقبة مستمرة لملفات الجورنال حسب نوع الصراف

**الخصائص:**
- مراقبة مجلدات الجورنال تلقائيًا
- استراتيجيات مخصصة لكل نوع صراف:
  - **NCR:** Overwrite Mirroring — يحتاج تتبع Offset دقيق
  - **GRG:** Daily Files — تتبع حسب ملف اليوم
  - **WN:** Daily Files — تتبع حسب ملف اليوم
- كشف الملفات الجديدة والمعدلة
- حساب MD5 Checksum لكل ملف
- ضغط وتشفير قبل الإرسال
- Polling احتياطي + FileSystemWatcher

#### ب) NetworkEngine (473 سطر)
**الوظيفة:** إدارة الاتصال الشبكي بين العميل والخادم

**الخصائص:**
- اتصال TCP/IP آمن مشفر (AES-256)
- Heartbeat تلقائي كل 30 ثانية
- إعادة اتصال ذكية (Exponential Backoff)
- ضغط البيانات (Deflate) قبل الإرسال
- دعم نقل الملفات الكبيرة (Chunked Transfer)
- دعم جميع أنواع الاتصال: LAN، ADSL، GSM/CDMA موديم

#### ج) ServerEngine (445 سطر)
**الوظيفة:** إدارة الخادم المركزي متعدد العملاء

**الخصائص:**
- خادم TCP متعدد العملاء (Multi-Client)
- إدارة الاتصالات المتزامنة
- استقبال وأرشفة الجورنال تلقائيًا
- إرسال أوامر بعيدة + Broadcast لجميع الصرافات
- إدارة حالة كل صراف (ATMInfo)

#### د) CommunicationProtocol (435 سطر)
**الوظيفة:** بناء وتحليل رسائل التراسل بين العميل والخادم

**أنواع الرسائل:**
```
HANDSHAKE | HEARTBEAT | JOURNAL_DATA | COMMAND | RESPONSE
FILE_TRANSFER | GHOST_FRAME | IMAGE_SYNC | BROADCAST
```

**عملية الأمان:**
```
البيانات الأصلية → ضغط Deflate → تشفير AES-256 → إضافة Checksum + MD5 → إرسال
```

#### هـ) TransactionAnalysisEngine (653 سطر)
**الوظيفة:** تحليل العمليات المالية من ملفات الجورنال

**يكشف عن:**
- عمليات السحب النقدي الناجحة والفاشلة
- استعلامات الرصيد
- البطاقات المحتجزة وكود السبب
- أخطاء وحدة الكاش (Cash Unit Errors)
- وضع المشرف (Supervisor Mode)
- عمليات إعادة التشغيل (Power Up Reset)
- أكواد الأخطاء: E3, M-18, M-02, M-03, M-05, M-10, M-11

**أزرار البحث المخصصة في واجهة القراءة:**
- Approved Transactions
- Power Up Reset
- Error E3
- Total Cash Error
- M-18
- M-02, M-03, M-05
- M-10, M-11

#### و) GhostRemoteEngine (296 سطر)
**الوظيفة:** الوصول عن بُعد للصراف "كشبح" (View Only)

**الخصائص:**
- لا يؤثر على الصراف ولا يُعيق العمليات
- لا يُسجل خروج المستخدم
- لا يقفل شاشة الصراف أمام العملاء
- التقاط شاشة مستمر (JPEG Streaming)
- جودة قابلة للتعديل من 10% إلى 100%

#### ز) ImageSyncEngine (296 سطر)
**الوظيفة:** نقل الصور إلى الصرافات وجدولة عرضها

**الخصائص:**
- نقل الصور الإعلانية والشعارات
- مزامنة ذكية (فقط الملفات المتغيرة)
- جدولة النقل (يوم محدد أو تاريخ)
- توزيع الصور من مجلد مشاركة على السيرفر حسب نوع الصراف
- العودة للعرض اليومي العادي بعد انتهاء مناسبة

#### ح) ReportExportEngine (341 سطر)
**الوظيفة:** تصدير التقارير بصيغ متعددة

**الصيغ المدعومة:**
- CSV: عمليات، تنبيهات، سجل مزامنة
- HTML: تقارير تحليلية احترافية
- (مقترح) PDF وExcel

## 4. بروتوكول التراسل بين العميل والخادم

### 4.1 تسلسل الاتصال الكامل

```
CLIENT (ATM)                                    SERVER (Host)
───────────────────────────────────────────────────────────
│                                                          │
│── EJLIVE_HANDSHAKE|ATM_ID|ATM_TYPE|VERSION|TIMESTAMP ──►│
│◄─ EJLIVE_ACK|SERVER_VERSION|SESSION_ID|TIMESTAMP ───────│
│                                                          │
│── HEARTBEAT|ATM_ID|TIMESTAMP ──────────────────────────►│ (كل 30 ثانية)
│◄─ HB_ACK|TIMESTAMP ─────────────────────────────────────│
│                                                          │
│── START_FILE|ATM_ID|FILENAME|SIZE|OFFSET|CHECKSUM ──────►│
│── CHUNK|SEQ_NUM|DATA ───────────────────────────────────►│
│◄─ CHUNK_ACK|SEQ_NUM ────────────────────────────────────│
│── CHUNK|SEQ_NUM|DATA ───────────────────────────────────►│ (يتكرر)
│── COMPLETE|FILENAME|TOTAL_CHECKSUM ────────────────────►│
│◄─ JOURNAL_ACK|FILENAME|VERIFIED ───────────────────────│
│                                                          │
│◄─ CMD_RESTART|CMD_ID|DELAY ────────────────────────────│
│── CMD_RESULT|CMD_ID|SUCCESS|RESULT_DATA ───────────────►│
│                                                          │
│◄─ GHOST_START_REQUEST|QUALITY ─────────────────────────│
│── GHOST_FRAME|JPEG_DATA ───────────────────────────────►│ (مستمر)
│◄─ GHOST_STOP ───────────────────────────────────────────│
│                                                          │
│◄─ IMAGE_SYNC|FILES_LIST|MD5_CATALOG ───────────────────│
│── IMAGE_SYNC_ACK|RECEIVED_FILES ───────────────────────►│
```

### 4.2 أنواع الأوامر البعيدة

| الأمر | الوظيفة | تأكيد مطلوب |
|-------|---------|------------|
| CMD_RESTART | إعادة تشغيل الصراف | نعم — تحذير مسبق |
| CMD_SHUTDOWN | إيقاف تشغيل الصراف | نعم |
| CMD_CHANGE_PASSWORD | تغيير كلمة سر الصراف | نعم |
| CMD_SCREENSHOT | لقطة شاشة فورية | لا |
| CMD_GHOST_START | بدء وضع الشبح | لا |
| CMD_GHOST_STOP | إنهاء وضع الشبح | لا |
| CMD_SEND_IMAGE | إرسال صورة واحدة | لا |
| CMD_SYNC_IMAGES | مزامنة جميع الصور | لا |
| CMD_FORCE_SYNC | إجبار إعادة مزامنة الجورنال | لا |
| CMD_GET_FILE | طلب ملف من الصراف | لا |
| CMD_SEND_FILE | إرسال ملف إلى الصراف | لا |

### 4.3 قواعد Idempotency (منع التكرار)

المفتاح الفريد لرفض التكرار:
```
ATM_ID + FileName + Offset + Checksum + SyncId
```

## 5. وصف تفصيلي لكل واجهات المستخدم

### 5.1 ClientMainForm.cs — تطبيق العميل (796 سطر)

**يُثبَّت على:** جهاز الصراف الآلي
**التبويبات الخمسة:**

#### تبويب 1: Connection (الاتصال)
- حقل عنوان IP للسيرفر
- رقم البورت (افتراضي: 5656)
- نوع الاتصال (LAN / ADSL / GSM / CDMA)
- زر "Connect" / "Disconnect"
- مؤشر حالة الاتصال (Connected/Disconnected)
- عرض Session ID
- عداد Heartbeat

#### تبويب 2: Synchronization (المزامنة)
- مسار مجلد الجورنال المصدري (Source Path) — مع زر استعراض
- مسار مجلد النسخ الاحتياطي المحلي (Local Backup Path) — مع زر استعراض
- حالة المزامنة الحالية (Timeline مقترحة):
  - **Source Read** → **Local Backup** → **Sent** → **Server Stored** → **Verified**
- شريط تقدم (Progress Bar)
- نسبة الإكمال، السرعة (KB/s)، الوقت المتبقي
- عداد: إجمالي الملفات / المزامنة / الفاشلة
- زر "Start Sync" / "Stop Sync"
- زر "Force Re-sync"

#### تبويب 3: Journal (عرض الجورنال)
- قراءة ملف الجورنال المحلي سطرًا بسطر
- خيارات الفلترة: حسب التاريخ، نوع العملية، كود الخطأ
- أزرار بحث سريع:
  - Approved Transactions
  - Power Up Reset
  - Error E3
  - Total Cash Error
  - M-18
  - M-02, M-03, M-05
  - M-10, M-11
- مربع بحث نصي حر

#### تبويب 4: Remote Commands (الأوامر البعيدة)
- استقبال وتنفيذ الأوامر الواردة من السيرفر
- عرض سجل الأوامر المستلمة
- تأكيد تنفيذ الأوامر بإشعار للسيرفر

#### تبويب 5: Settings & Log (الإعدادات والسجل)
- نوع الصراف (NCR / GRG / WN)
- ATM ID
- إعدادات الاتصال
- إعدادات جودة الشبكة (LAN/ADSL/CDMA/GSM)
- سجل الأحداث (Log Viewer) مع فلترة المستويات
- زر تصدير اللوج
- Splash Screen عند الفتح

### 5.2 ServerMainForm.cs — تطبيق الخادم (731 سطر)

**يُشغَّل على:** الخادم المركزي (Host Server)
**التبويبات الستة:**

#### تبويب 1: Server — لوحة NOC

**لوحة المقاييس (8 مقاييس):**
1. إجمالي الصرافات المتصلة
2. إجمالي الصرافات المتوقفة عن الإرسال
3. حجم الجورنال المستلم اليوم
4. معدل نجاح المزامنة (%)
5. عدد التنبيهات النشطة
6. سرعة الاستقبال الكلية (KB/s)
7. آخر وقت استلام بيانات
8. مساحة التخزين المتبقية على السيرفر

**بطاقات الصرافات (ATM Cards) — بنظام الألوان:**
لكل صراف بطاقة تعرض:
- معرف الصراف (ATM ID) ونوعه
- نوع الاتصال (LAN/CDMA/GSM)
- الحالة بنظام الألوان (انظر القسم 6)
- آخر رسالة جورنال مستلمة
- آخر خطأ في وحدة الكاش (أحمر اللون)

## 7. متطلبات النظام الوظيفية والتقنية

### 7.1 متطلبات المزامنة الثلاثية الأطراف

```
المصدر (الصراف) ──► النسخة الاحتياطية المحلية ──► السيرفر (Host)
      ↑                       ↑                        ↑
  مراقبة مستمرة          تزامن لحظي               أرشفة آمنة
```

**قواعد المزامنة:**
1. النسخ الاحتياطي يعكس المصدر لحظيًا
2. لا يُرسل للسيرفر إلا ما تم التحقق منه في النسخة الاحتياطية
3. فحص ثلاثي مستمر: المصدر ↔ النسخة الاحتياطية ↔ السيرفر
4. استرجاع الملفات القديمة (ما قبل تثبيت البرنامج) تلقائيًا

### 7.2 متطلبات الاستمرارية والتعافي

- التشغيل التلقائي مع Windows كخدمة ذات امتيازات مدير
- إعادة اتصال تلقائية عند انقطاع الشبكة
- Watchdog يُعيد تشغيل البرنامج عند التجمد أو التوقف
- حفظ حالة المزامنة في قاعدة بيانات محلية (ليس في الذاكرة فقط)
- Client Outbox Queue لضمان عدم فقدان البيانات عند الانقطاع

### 7.3 متطلبات الأمان

- تشفير AES-256 لجميع البيانات المنقولة
- تشفير بيانات الجورنال أثناء التخزين (ليس فقط النقل)
- Audit Log غير قابل للتعديل لكل أمر حساس
- صلاحيات مستخدمين: مراقب / مدير نظام / مدقق / دعم فني
- تحقق MD5/SHA-256 عند كل نقطة اتصال
- تحذير قبل تنفيذ أي أمر حساس (Restart, Shutdown, Ghost)

### 7.4 متطلبات الأداء

- استيعاب مئات الرسائل لحظيًا من شبكة الصرافات
- لا يؤثر على أداء الصراف أثناء التشغيل
- التشغيل الصامت في الخلفية
- دعم حتى 7+ صرافات في قائمة الفحص (قابل للزيادة)

```
1. Persistent Sync State (SQLite):
   حقول: SyncId, ATM_ID, FileName, Offset, Checksum, State, RetryCount
   
2. Client Outbox Queue:
   حفظ الجورنال في طابور محلي قبل الإرسال
   
3. Idempotency على السيرفر:
   رفض التكرار بـ: ATM_ID + FileName + Offset + Checksum + SyncId
   
4. Offset Tracking حقيقي:
   NCR: تتبع EJDATA.LOG, EJRCPY.LOG, EJDATA.LOb بـ Byte Offset
   GRG/WN: تتبع بـ ملف اليوم
   
5. File Locking Fix:
   FileStream مع FileShare.ReadWrite
   Exponential Backoff: 100ms → 500ms → 1s
```

#### الحزمة الثانية — المعمارية
```
6. Producer-Consumer Pattern في ServerEngine:
   BlockingCollection<T> أو Channel<T>
   معالجة منفصلة عن خيط الاستلام
   
7. Async/Await الكامل في NetworkEngine
   
8. Length-Prefixed Framing للنقل:
   START_FILE → CHUNK → CHUNK_ACK → COMPLETE → VERIFY
   
9. Exponential Backoff + Jitter للإعادة:
   5s → 10s → 30s → 60s + عشوائية لمنع الاندفاع
   
10. SHA-256 Hash للتحقق النهائي من كل ملف
```

### 10.2 تدفق البيانات الكامل (Data Flow)

```
[ATM Dispenses Cash]
        ↓
[ATM Software writes to EJDATA.LOG / EJ_date.dat]
        ↓
[EJLive.Client — FileWatcherEngine detects change]
        ↓
[Reads new bytes from FileOffset (not from start)]
        ↓
[Saves to LocalBackup folder (verified copy)]
        ↓
[Adds to OutboxQueue (persistent)]
        ↓
[NetworkEngine: Compress → Encrypt AES-256 → Checksum]
        ↓
[TCP/5656: START_FILE → CHUNK × N → COMPLETE]
        ↓
[EJLive.Server — ServerEngine receives into Channel<T>]
        ↓
[Processor: Decrypt → Decompress → Verify Checksum]
        ↓
[ArchiveManager: Save to ATM folder / SQLite index]
        ↓
[TransactionAnalysisEngine: Parse journal entries]
        ↓
[MonitoringDashboard: Update ATM card color & stats]
        ↓
[ReportExportEngine: Available for reporting]
```

## ملاحظة ختامية

هذه الوثيقة تمثل **المرجع الموحد الكامل** لنظام EJLive Enterprise، مُستخلَصة من تحليل معمق لأكثر من **30 وثيقة وملف** شاملة وثائق المعمارية، وثائق الفجوات، وثائق التحسينات، متطلبات المزامنة، قوائم الفحص، وتوثيق الإصدارات من v3.2.1 إلى v3.4.0.

النظام يمتلك **أساسًا برمجيًا قويًا** في التشفير ومعالجة الجورنال. أولوية التطوير الفورية هي:
1. سد فجوات الموثوقية الجوهرية (الحزمة الأولى)
2. تحسين المعمارية للتوسع (الحزمة الثانية)
3. رفع الجودة البصرية لمستوى NOC احترافي (الحزمة الثالثة)

هذا سيحوّل EJLive من "نظام يعمل" إلى **"بنية تحتية مصرفية موثوقة"** قادرة على المنافسة عالميًا.
